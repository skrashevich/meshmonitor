MeshMonitor **4.0** is on the horizon. The headline change is **Multi-Source** support — a single MeshMonitor will be able to connect to **multiple Meshtastic nodes at once**, each with its own Virtual Node, auto-responder, scheduler, and per-source permissions.

Preview what's changing at [meshmonitor.org/features/multi-source](https://meshmonitor.org/features/multi-source).

## Before you upgrade

4.0 ships extensive schema migrations and a reorganized permission model. **Plan to review:**

- **Configuration** — several environment variables (including the Virtual Node env vars) are removed in 4.0; per-source settings move to the UI (see [Multi-Source](https://meshmonitor.org/features/multi-source))
- **Security / Permissions** — the permission matrix becomes per-source; every user's access on every source will need a fresh review (see [Per-Source Permissions](https://meshmonitor.org/features/per-source-permissions))

**Back up your system before upgrading.** Use Settings → System Backup or copy your `/data` volume.
