MeshMonitor v3.3.0 introduces **user-specific permissions for Virtual Channels**, along with several quality-of-life improvements.

## Virtual Channel Permissions

Virtual channels (from the Channel Database) now support per-user **View Map** and **Read** permissions, just like device channels.

## Action Required

Administrators should review their user permissions for any virtual channels:

1. Navigate to **Settings > Users** and select a user
2. Scroll to the new **Virtual Channel Permissions** section
3. For each virtual channel, verify the appropriate **View Map** and **Read** permissions are assigned
4. Non-admin users will only see nodes and messages from virtual channels they have been granted access to

## Other Improvements

- **Emoji tapback reactions** - Replying to a message with just an emoji now renders as a tapback pill instead of a full message bubble
- **Sticky/pinned nodes** in the Messages tab
- **Map improvements** - Clicking a node with Show Traceroute enabled now zooms to the node if no traceroute data exists
- **Link Quality chart** improvements for sparse data
- **Channel fixes** - Correct encryption status display, preserved disabled channels, shorthand PSK support in Channel Database
